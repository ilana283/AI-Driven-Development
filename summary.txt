--- English (handoff for another developer) ------------------------------------

1. PROJECT OVERVIEW
   - Stack: Next.js 16.x, React 19, TypeScript, Tailwind CSS v4, ESLint.
   - Repo: GitHub (e.g. ilana283/my-next-app). Hosting: Vercel linked to GitHub.
   - Site language: English. Single-page portfolio (sections + anchor nav).

2. WHAT THE OWNER ASKED FOR (requirements collected over time)
   - Portfolio site reflecting resume: System Integration Engineer, EE background.
   - English only; no hobbies section.
   - Design: not “plain resume on the web” — use card/panel boxes and subtle color
     (sky accent borders, gradient background) similar in spirit to example sites
     discussed (e.g. harelcoman.dev) — implemented as reusable Panel components.
   - Git workflow: do NOT push unreviewed changes straight to production habitually;
     use a branch per change, open a Pull Request, merge when satisfied. Vercel
     Preview for the branch; Production typically tracks main.
   - After git stash / pull / stash pop: resolve merge conflicts carefully — never
     commit files containing conflict markers (<<<<<<<, =======, >>>>>>>); they
     break `next build` and Vercel deployments.
   - Explanations for course context: prefer simple language when describing
     tooling (owner is EE background, not primarily SW engineering).

3. REPOSITORY LAYOUT (important paths)
   - app/site.ts       — All structured content: nav, about[], experience[],
                         education[], military, languages, projects[]. Edit here
                         for text/resume updates without restructuring UI.
   - app/page.tsx      — UI: Header, sections, Panel + SectionTitle, footer.
   - app/layout.tsx    — Root layout, Geist fonts, metadata (title/description).
   - app/globals.css   — Tailwind import, CSS variables, body font.
   - public/           — Static assets if added later.

4. LOCAL DEVELOPMENT
   - Prerequisites: Node.js, npm (or pnpm if you switch).
   - From repo root (folder that contains package.json and .git):
       npm install
       npm run dev
     Open http://localhost:3000
   - Quality checks:
       npm run lint
       npm run build

5. GIT & DEPLOYMENT WORKFLOW (agreed process)
   - git checkout main && git pull origin main
   - git checkout -b feature/short-description
   - Make changes, then:
       git add .
       git commit -m "Clear message"
       git push -u origin feature/short-description
   - On GitHub: open Pull Request into main; verify Vercel Preview in PR comment.
   - Merge PR when ready → production redeploy (per Vercel project settings).
   - Locally after merge: git checkout main && git pull origin main
   - If pull is blocked by local edits: git stash, pull, git stash pop — then
     resolve conflicts in editor before committing.

6. KNOWN ISSUES ALREADY FIXED (historical)
   - Unresolved merge conflict markers left inside app/page.tsx after stash pop
     caused Turbopack/Next build to fail; Vercel deployment failed for same reason.
     Fix: remove markers and keep a single coherent page.tsx (card layout version).

   - Any local uncommitted changes or unpushed branches are not tracked here;
